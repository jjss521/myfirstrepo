## 一、需求概述

### 1.1 目标
构建一套"前端管理 + 后端存储 + Python生成引擎"的软件系统，实现：
1. **上传管理**：上传日常工作积累的可研文本、初步设计文本、负荷计算书等参考文件
2. **分类归档**：按给水工程、排水工程、环卫工程、道路工程的电气/自控说明深度要求，自动分类分栏保存并制作成模板
3. **智能生成**：输入负荷计算书Excel，根据规范深度要求和预设模板，自动生成完整的电气自控说明 + 设备材料表

### 1.2 用户角色
- 贾瑟（电气工程师）— 唯一用户，本地部署使用

## 四、数据库设计

### 4.1 核心表结构

```sql
-- 工程类型表
CREATE TABLE project_types (
    id INTEGER PRIMARY KEY,
    code VARCHAR(20) UNIQUE NOT NULL,  -- water_supply, drainage, road, sanitation
    name VARCHAR(50) NOT NULL,          -- 给水工程、排水工程、道路工程、环卫工程
    design_stage VARCHAR(20) NOT NULL,  -- 可研, 初步设计, 施工图
    description TEXT
);

-- 栏目表（每个工程类型下的电气/自控栏目）
CREATE TABLE sections (
    id INTEGER PRIMARY KEY,
    project_type_id INTEGER REFERENCES project_types(id),
    category VARCHAR(20) NOT NULL,      -- 电气, 自控, 照明
    section_order INTEGER NOT NULL,     -- 栏目序号
    title VARCHAR(100) NOT NULL,        -- 栏目名称
    depth_requirement TEXT NOT NULL,    -- 深度要求（来自2025规范）
    default_content TEXT,               -- 预设模板内容
    is_required BOOLEAN DEFAULT 1       -- 是否为必填栏目
);

-- 模板表
CREATE TABLE templates (
    id INTEGER PRIMARY KEY,
    project_type_id INTEGER REFERENCES project_types(id),
    name VARCHAR(100) NOT NULL,
    file_path VARCHAR(500),             -- 模板文件路径
    content_type VARCHAR(20),           -- 电气说明, 自控说明, 设备材料表
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 参考文档表
CREATE TABLE documents (
    id INTEGER PRIMARY KEY,
    project_type_id INTEGER REFERENCES project_types(id),
    name VARCHAR(200) NOT NULL,
    file_path VARCHAR(500) NOT NULL,
    doc_type VARCHAR(20),              -- 可研文本, 初设文本, 负荷计算书, 其他
    tags TEXT,                          -- JSON数组，如 ["给水","10kV","水泵"]
    uploaded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 生成记录表
CREATE TABLE generation_logs (
    id INTEGER PRIMARY KEY,
    project_type_id INTEGER REFERENCES project_types(id),
    input_file VARCHAR(500),           -- 输入的负荷计算书
    output_file VARCHAR(500),          -- 输出文件
    template_used VARCHAR(100),        -- 使用的模板
    generated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(20)                 -- success, failed
);
```

## 五、后端模块设计

### 5.1 技术栈
- **框架**：FastAPI（Python 3.11+）
- **数据库**：SQLite（本地单用户，无需MySQL/PostgreSQL）
- **文件处理**：python-docx（Word模板）、openpyxl（Excel负荷计算）
- **前端**：Vue 3 + Element Plus（SPA单页应用）

### 5.2 API接口设计

```
POST   /api/documents/upload          # 上传参考文档
GET    /api/documents                 # 文档列表（可按类型筛选）
DELETE /api/documents/{id}            # 删除文档

GET    /api/sections                  # 获取栏目列表（按工程类型+阶段）
POST   /api/sections                  # 新增/编辑栏目
PUT    /api/sections/{id}             # 更新栏目内容

GET    /api/templates                 # 模板列表
POST   /api/templates/create          # 从参考文档创建模板
PUT    /api/templates/{id}            # 编辑模板
DELETE /api/templates/{id}            # 删除模板

POST   /api/generate                  # 提交负荷计算书，生成设计文件
GET    /api/generate/logs             # 生成记录
GET    /api/generate/download/{id}    # 下载生成的文件
```

## 六、Python生成引擎设计

### 6.1 核心流程

```
输入: 负荷计算书Excel
  │
  ├── 步骤1: 解析Excel
  │   ├── 识别设备类型、容量、数量
  │   ├── 提取需要系数、功率因数
  │   └── 计算：总容量、计算负荷、无功补偿、变压器容量
  │
  ├── 步骤2: 匹配工程类型
  │   ├── 从Excel表头/用户选择确定工程类型（给水/排水/道路/环卫）
  │   └── 加载对应的栏目深度要求+模板
  │
  ├── 步骤3: 按栏目逐一生成内容
  │   ├── 电气设计部分

  │   │   ├── 电源（从模板+Excel参数）

  │   │   ├── 供配电系统（从模板获取）

  │   │   ├── 保护和控制（从模板获取）
  │   │   ├── 计量（从模板获取）
  │   │   ├── 管线敷设（从模板获取）
  │   │   ├── 照明（从模板获取）
  │   │   ├── 设备选型（结合Excel参数）
  │   │   └── 防雷接地（从模板获取）
  │   │
  │   ├── 自控/弱电部分
  │   │   ├── 自控系统
  │   │   ├── 仪表系统
  │   │   ├── 工业电视/安防/通信
  │   │   ├── 火灾自动报警
  │   │   └── 防雷接地/管缆敷设
  │   │
  │   └── 设备材料表

  └── 步骤4: 输出Word文档
```

### 6.2 负荷计算Excel解析规则

| 序号 | 序号 | 1 |
| 设备名称 | 用电设备名称 | 取水泵 |
| 单台功率(kW) | 单台额定功率 | 110 |
| 数量 | 设备台数 | 3 |
| 工作数量 | 同时工作台数 | 2 |
| 备用数量 | 备用台数 | 1 |
| 需要系数(Kx) | 需要系数 | 0.8 |
| 功率因数(cosφ) | 功率因数 | 0.85 |
| 安装位置 | 所属车间/区域 | 取水泵房 |
| 电压等级(V) | 供电电压 | 380 |
| 备注 | 备注 | 变频控制 |

**自动计算输出：**
- 安装容量(kW) = 单台功率 × 数量
- 工作容量(kW) = 单台功率 × 工作数量
- 计算有功(kW) = 工作容量 × Kx
- 计算无功(kvar) = 计算有功 × tan(arccos(cosφ))
- 视在功率(kVA) = sqrt(有功² + 无功²)
- 计算电流(A) = 视在功率 / (√3 × 电压)
- 总计算负荷汇总 → 变压器容量选择
- 无功补偿容量 → 补偿后功率因数

## 十、关键技术决策

### 10.1 为什么选择SQLite？
- 单用户本地部署，无需数据库服务器
- 零配置，数据文件便携
- 足够支持本项目的数据量

### 10.2 为什么选择python-docx？
- 熟练操作Word .docx格式
- 支持模板填充、表格生成、样式设置
- 与电气工程师的工作流（最终交付Word文档）直接匹配

### 10.3 为什么命令行版和Web版双模式？
- **命令行版**：快速、直接在工程师电脑上运行，适合批量生成
- **Web版**：可视化管理模板和文档，适合日常积累和维护
