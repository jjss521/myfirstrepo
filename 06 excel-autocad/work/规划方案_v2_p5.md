## 数据模型定义 (data_model.py)

```python
from dataclasses import dataclass, field
from typing import Optional, List, Dict, Tuple
from enum import Enum


class LoadType(str, Enum):
    POWER = "动力"
    LIGHTING = "照明"
    VFD = "变频"
    AC = "空调"
    SOCKET = "插座"
    SPARE = "备用"
    CAPACITOR = "电容补偿"


# ---- 数据层 ----

@dataclass
class RawRow:
    """Excel原始行数据"""
    row_number: int
    values: Dict[str, str]

@dataclass
class CircuitRecord:
    """校验后的回路记录"""
    row_number: int
    circuit_id: str
    circuit_name: str
    load_type: LoadType
    rated_power_kw: float
    rated_current_a: float
    breaker_model: str
    breaker_poles: int
    breaker_trip_current_a: float
    ct_ratio: str
    cable_type: str
    cable_section: str
    vfd_model: Optional[str] = None
    vfd_power_kw: Optional[float] = None
    remark: Optional[str] = None

@dataclass
class ErrorRecord:
    """校验失败记录"""
    row_number: int
    circuit_id: str
    error_type: str
    error_message: str
    raw_values: Dict[str, str]


# ---- 映射层 ----

@dataclass
class CircuitWithBlock:
    """关联了图块的回路"""
    record: CircuitRecord
    block_name: str
    attributes: Dict[str, str] = field(default_factory=dict)

@dataclass
class BlockDefinition:
    """图块定义(从block_catalog读取)"""
    name: str
    description: str
    applicable: Dict[str, str]
    attributes: List[str]

class BlockCatalog:
    blocks: List[BlockDefinition]
    def find(self, block_name: str) -> Optional[BlockDefinition]: ...


# ---- 布局层 ----

@dataclass
class PaperSize:
    name: str
    width: float
    height: float

@dataclass
class Placement:
    block_name: str
    x: float
    y: float
    attributes: Dict[str, str]
    circuit_id: str

@dataclass
class BusLine:
    x: float
    y_start: float
    y_end: float

@dataclass
class GroupLabel:
    text: str
    x: float
    y: float

@dataclass
class LayoutResult:
    placements: List[Placement]
    bus_line: BusLine
    group_labels: List[GroupLabel]
    paper_size: PaperSize


# ---- 配置模型 ----

@dataclass
class AppConfig:
    autocad: AcadConfig
    block_library: BlockLibraryConfig
    excel: ExcelConfig
    block_mapping: BlockMappingConfig
    layout: LayoutConfig
    sort: SortConfig
    output: OutputConfig
    text_style: TextStyleConfig
    title_block: TitleBlockConfig
# 各子配置类与config.yaml结构一一对应,详见配置文件节
```

## 异常处理与日志

### 异常分类与处理策略

| 异常类型 | 处理策略 | 是否退出 |
|---|---|---|
| Excel文件不存在 | 打印错误 -> 退出 | 是 |
| Sheet未找到 | 列出可用Sheet -> 退出 | 是 |
| 必填列匹配率<70% | 列出已识别列 -> 退出 | 是 |
| 全部回路校验失败 | 打印汇总 -> 退出 | 是 |
| 部分回路校验失败 | 跳过,继续,报告中列出 | 否 |
| 图块库DWG不存在 | 打印错误 -> 退出 | 是 |
| 图块库中有缺失图块 | 打印警告 -> 退出 | 是 |
| AutoCAD未运行 | 提示启动 -> 退出 | 是 |
| AutoCAD版本不兼容 | 列出支持版本 -> 退出 | 是 |
| 单个图块插入失败 | 跳过该回路,记录错误,继续 | 否 |
| 属性写入失败 | 记录警告,继续 | 否 |

核心原则: 运行环境类问题直接退出; 单回路数据问题跳过继续,确保尽可能多的回路被处理。

## 命令行接口设计

```
用法:
  pdsg <excel_path> [OPTIONS]

参数:
  excel_path           Excel回路清单文件路径(.xlsx),必填

选项:
  -c, --config PATH    配置文件路径(默认./config.yaml)
  -o, --output PATH    输出DWG路径(覆盖配置文件设置)
  --dry-run            仅读取校验,不连接AutoCAD,不生成DWG
  --log-level LEVEL    日志级别: DEBUG|INFO|WARNING|ERROR(默认INFO)
  --version            显示版本信息
  -h, --help           显示帮助

示例:
  pdsg "D:\项目\配电柜回路表.xlsx"
  pdsg "回路表.xlsx" -c "my_config.yaml" --dry-run
  pdsg "回路表.xlsx" -o "D:\输出\系统图.dwg" --log-level DEBUG
```

## 风险与缓解

| # | 风险 | 概率 | 影响 | 缓解措施 |
|---|---|---|---|---|
| R1 | AutoCAD COM接口版本行为不一致 | 中 | 高 | ProgID探测+运行时版本检测+日志记录版本 |
| R2 | 图块库DWG缺少必需属性Tag | 高 | 中 | 启动时校验block_catalog vs 实际DWG |
| R3 | Excel格式与预期不符 | 中 | 中 | 三级模糊匹配+标准模板+错误报告指出不匹配列 |
| R4 | 200回路时pyautocad性能瓶颈 | 低 | 低 | <=200时COM调用足够;超阈值引入批量操作 |
| R5 | AutoCAD未启动或版本不对 | 中 | 中 | 启动检测+明确提示支持版本和操作步骤 |
| R6 | 图纸幅面自适应不合理 | 低 | 低 | 最小幅面限制+允许手动指定幅面 |
| R7 | 变频回路图块缺VFD属性 | 中 | 低 | block_catalog校验+写入前检查Tag存在性 |

## 关键决策记录

| # | 决策 | 结论 | 依据 |
|---|---|---|---|
| D1 | 图块粒度 | 回路级(一整回路一个图块) | 用户确认;减少插入次数,保证内部符号相对位置准确 |
| D2 | 互操作方案 | pyautocad (COM) | Python生态无缝,<=200回路性能足够 |
| D3 | 配置文件格式 | YAML | 可读性优,支持注释,适合电气工程师编辑 |
| D4 | 列名匹配 | 三级模糊匹配 | 适配不同设计院Excel命名习惯 |
| D5 | 幅面选择 | 自动+手动覆盖 | 兼顾智能与灵活性 |
| D6 | 错误处理 | 部分失败不中断 | 单回路错不应阻塞全局 |
| D7 | 测试策略 | 分层,AutoCAD全mock | CI可运行,不依赖AutoCAD环境 |
| D8 | 打包 | PyInstaller | 用户无需安装Python |
