# 配电柜系统图自动生成程序 -- 规划方案 V2
## Part 1: 概述、架构、技术选型、图块库设计

### 1.1 项目定义

- 项目名称: Excel-AutoCAD 配电柜系统图自动生成工具
- 简称: PDSG (Power Distribution Single-line Generator)
- 版本: V1.0
- 目标用户: 电气设计工程师

### 1.2 业务背景

电气设计工程师在设计 400V 低压配电柜时,需根据《配电柜回路清单》Excel 表格,在 AutoCAD 中逐个回路绘制系统单线图。典型项目包含几十至两百个回路,每个回路需在图纸上放置对应的回路图块并填写断路器型号、电缆规格、负荷参数、电流互感器变比等技术参数。手工操作耗时 2-4 小时且易因疏忽出错。

### 1.3 目标

导入配电柜各回路的 Excel 表格(含回路编号、断路器型号、电缆型号规格、负荷值、额定电流、变频器型号、CT 参数等),在 AutoCAD 中用预先设置好的回路级图块一一对应进行拼接,生成完整的 400V 配电系统单线图。

### 1.4 核心约束(已确认)

| 约束项 | 值 |
|---|---|
| 目标 AutoCAD 版本 | AutoCAD 2020 / 2022 |
| 图块粒度 | 每个回路一个完整图块(含母排接入点、断路器、CT、变频器位置、电缆接线端) |
| 回路数量 | 起步数十个,上限约200回路 |
| 电压等级 | 400V 低压配电系统 |
| 图纸幅面 | 自动选择(A2/A1/A0),可手动指定 |
| 运行方式 | CLI 命令行工具,单文件处理 |

### 1.5 范围边界

范围内:
- Excel 读取与校验
- 回路图块自动选择
- 图块属性批量填写
- 母线自动绘制
- 回路图块沿母线自动排列
- 图框与标题栏插入
- 运行日志与错误报告生成

范围外:
- 三维布柜图生成(仅单线系统图)
- 负荷计算(由外部软件完成)
- 图块本身的电气设计(需电气工程师在 AutoCAD 中预建)
- 多人协同编辑

### 2.1 分层架构

程序采用五层架构:

输入层: Excel回路清单(.xlsx) + 回路图块库(block_library.dwg) + 配置文件(config.yaml)
数据层: excel_reader.py(读取/校验) + data_model.py(数据类)
业务逻辑层: block_mapper.py(回路图块映射) + layout_engine.py(坐标计算) + attribute_builder.py(属性组装)
绘图引擎层: block_library.py(图块库管理) + cad_drawer.py(AutoCAD COM互操作)
输出层: 配电系统图(output.dwg) + 处理报告(report.html) + 运行日志(app.log)

### 2.2 数据流(端到端)

Step 1: CLI 接收 Excel 路径、图块库路径、(可选)配置文件路径
Step 2: excel_reader 打开 Excel, 定位 Sheet, 列名映射, 逐行校验, 输出 List[CircuitRecord]
Step 3: block_mapper 读取映射规则, 逐回路匹配图块名, 输出 List[CircuitWithBlock]
Step 4: layout_engine 计算坐标, 排序, 可选分组, 自动选幅面, 输出 LayoutResult
Step 5: cad_drawer 连接 AutoCAD, 导入图块库, 插入图块, 填写属性, 绘制母线, 插入图框, 保存 DWG
Step 6: reporter 汇总成功/警告/失败, 输出 HTML 报告

### 3.1 技术栈

| 层次 | 选型 | 版本要求 | 选型理由 |
|---|---|---|---|
| 语言 | Python | 3.10+ | Excel与COM互操作生态成熟 |
| Excel | openpyxl + pandas | >=3.0 | 处理.xlsx/合并单元格/批量清洗 |
| AutoCAD | pyautocad (底层comtypes) | >=0.6 | 轻量ActiveX/COM封装 |
| 配置 | PyYAML | >=6.0 | 可读性好,支持注释 |
| 报告 | jinja2 | >=3.1 | HTML模板生成 |
| 测试 | pytest + pytest-mock | >=7.0 | 逻辑层可脱离AutoCAD测试 |
| 打包 | PyInstaller | >=5.0 | 一键.exe,用户无需装Python |

### 3.2 AutoCAD COM版本适配

| AutoCAD 版本 | ProgID |
|---|---|
| 2020 | AutoCAD.Application.23.1 |
| 2022 | AutoCAD.Application.24.1 |

启动时按优先级探测 ProgID 列表,取首个可连接的版本。配置文件也可指定优先版本。

### 3.3 不选方案对比

| 方案 | 不选原因 |
|---|---|
| .NET SDK (C#) | 开发周期长,调试不便,与Excel生态隔离 |
| AutoLISP / DCL | Excel处理极弱,无数据清洗能力 |
| VBA (AutoCAD内嵌) | 版本兼容性差,Excel处理受限 |

### 4. 图块库设计规范

### 4.1 图块粒度定义(关键设计决策)

已确认: 一个图块 = 一个完整配电回路,包含: 母排接入点(图块顶部,插入基准点), 电流互感器(CT), 断路器(QF), 变频器(VFD,仅变频回路), 电缆出线端, 负荷标注区域。

### 4.2 图块属性定义

必填属性:

| 属性 Tag | 说明 | Excel来源 |
|---|---|---|
| CIRCUIT_ID | 回路编号 | 回路编号 |
| CIRCUIT_NAME | 回路名称 | 回路名称 |
| BREAKER_MODEL | 断路器型号 | 断路器型号 |
| BREAKER_POLES | 极数(1P/2P/3P/4P) | 断路器极数 |
| BREAKER_Ir | 脱扣整定电流(A) | 脱扣器电流 |
| CABLE_TYPE | 电缆型号 | 电缆型号 |
| CABLE_SECTION | 电缆规格 | 电缆规格 |
| LOAD_POWER | 负荷功率(kW) | 额定功率 |
| LOAD_CURRENT | 计算电流(A) | 额定电流 |
| LOAD_TYPE | 负荷类型 | 负荷类型 |
| CT_RATIO | CT变比 | CT变比 |

可选属性:

| 属性 Tag | 条件 | 说明 | Excel来源 |
|---|---|---|---|
| VFD_MODEL | 变频回路 | 变频器型号 | 变频器型号 |
| VFD_POWER | 变频回路 | 变频器功率(kW) | 变频器功率 |
| REMARK | 任意 | 备注 | 备注 |

### 4.3 图块命名规范

格式: LOOP_<负荷类型缩写>_<极数>_<序号>

示例: LOOP_PWR_3P_01 = 三相动力回路 Type 01

负荷类型缩写: PWR=动力, LGT=照明, VFD=变频, AC=空调, SKT=插座, SPR=备用, CAP=电容补偿

### 4.4 图块插入点

每个回路图块的插入基准点统一为母排接入点(图块顶部母线连接处)。沿垂直母线等距排列时各回路自然对齐。

### 4.5 图块库文件组织

blocks/ 目录包含:
- block_library.dwg -- 主图块库(电气工程师创建)
- title_block.dwg -- 图框/标题栏
- block_catalog.yaml -- 图块目录(程序读取)
- README.md -- 图块创建规范

block_catalog.yaml 示例:

```yaml
blocks:
  - name: "LOOP_PWR_3P_01"
    description: "三相动力回路(标准型)"
    applicable:
      load_type: "动力"
      poles: 3
    attributes: [CIRCUIT_ID, CIRCUIT_NAME, BREAKER_MODEL, BREAKER_POLES,
                 BREAKER_Ir, CABLE_TYPE, CABLE_SECTION, LOAD_POWER,
                 LOAD_CURRENT, LOAD_TYPE, CT_RATIO]
  - name: "LOOP_VFD_3P_01"
    description: "三相变频回路"
    applicable:
      load_type: "变频"
      poles: 3
    attributes: [CIRCUIT_ID, CIRCUIT_NAME, BREAKER_MODEL, BREAKER_POLES,
                 BREAKER_Ir, CABLE_TYPE, CABLE_SECTION, LOAD_POWER,
                 LOAD_CURRENT, LOAD_TYPE, CT_RATIO, VFD_MODEL, VFD_POWER]
```
