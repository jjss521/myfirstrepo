# V2G（Vehicle to Grid）供电部门/电网侧相关标准协议汇总

> 整理时间：2026-06-24  
> 适用范围：中国国家电网/南方电网的V2G项目，从国际到国内各层级的标准体系

---

## 一、分层架构总览

V2G 涉及 **4 个层次的协议栈**，每一层对应不同部门的职责边界：

```
┌─────────────────────────────────────────────────────┐
│  第4层：电力市场/交易层                                │
│  国家发改委/能源局政策 + 虚拟电厂平台                    │
├─────────────────────────────────────────────────────┤
│  第3层：电网调度/聚合层 (Utility/Aggregator Level)     │
│  IEEE 2030.5 / 国网企标 / 南网企标                      │
├─────────────────────────────────────────────────────┤
│  第2层：充电基础设施管理层 (Charging Station Level)    │
│  OCPP 1.6J / OCPP 2.0.1 / GB/T 28142                  │
├─────────────────────────────────────────────────────┤
│  第1层：车-桩通信层 (Vehicle-Edge Level)               │
│  ISO 15118-20 / GB/T 27930-2023                        │
├─────────────────────────────────────────────────────┤
│  物理并网与安全保护                                    │
│  IEEE 1547-2018 / GB/T 33593 / 国网反孤岛保护           │
└─────────────────────────────────────────────────────┘
```

## 二、国际标准体系

### 2.1 ISO 15118 系列（车-桩通信层）

| 分标准 | 名称 | 职责 |
|--------|------|------|
| ISO 15118-2 | 交流/直流单向充电 | 基础充电会话管理、功率协商 |
| ISO 15118-20 | **双向充电/放电 + 交流直流统一** | 最新版本，支持 V2G 功率反向流动、动态价格响应 |
| ISO 15118-21 | PLCA（PHY层适配） | 以太网物理层 |

**中国对接情况**：中国目前国标充电通信协议 GB/T 27930 主要对应 ISO 15118-2，V2G 功能正在向 GB/T 27930-2023 和 ISO 15118-20 兼容过渡。

### 2.2 IEEE 1547-2018（并网安全层）

| 关键条款 | 要求 |
|----------|------|
| §6.4.2 | V2G 必须具备 **防孤岛（Anti-Islanding）** 保护 |
| §6.4.3 | 电压/频率穿越能力（ ride-through ） |
| §6.4.5 | 电能质量（谐波、电压波动、闪变） |
| §6.5 | 反向有功/无功功率控制 |
| §7.2 | 与配电系统保护装置（继电器、FRT）的配合 |

**供电部门视角**：这是电网公司审批 V2G 设备并网的 **强制性电气安全前提**。

### 2.3 IEEE 2030.5 / SEP 2.0（电网调度层）

| 角色 | 说明 |
|------|------|
| 适用方 | **电网公司 / 聚合商（Aggregator） ↔ EMS 站点控制器** |
| 协议 | RESTful HTTP + CoMPaaS（CoAP 扩展） |
| 功能 | 事件驱动控制（削峰填谷调度）、动态电价信号、遥测反馈 |
| 计费验证 | 提供可验证的计量数据，支撑电力市场结算 |

**供电部门视角**：IEEE 2030.5 是美国 CAISO 等电网公司主推的 V2G 调度协议，用于电网向聚合商下发需求响应（DR）事件。

### 2.4 OCPP（充电管理层）

| 版本 | 差异 | V2G 相关 |
|------|------|----------|
| OCPP 1.6J | 最新版本，J 扩展支持 ISO 15118 | Smart Charging Profile（充电配置）、Remote Start/Stop |
| OCPP 2.0.1 | 新增 Charging Schedule V2G 扩展 | 支持 V2G 充电计划、电价事件、合同管理 |

**供电部门视角**：OCPP 不是电网调度协议，而是充电运营平台对充电桩的管理协议。V2G 场景下，电网调度信号先到聚合商，聚合商通过 OCPP 把充电计划下发到桩。

## 三、中国国家标准体系

### 3.1 国标（GB/T）核心标准

| 标准号 | 名称 | 层级 | 版本 | 说明 |
|--------|------|------|------|------|
| **GB/T 27930-2023** | 非车载传导式充电机与电动汽车之间的数字通信协议 | 车-桩层 | 2023修订版 | 替代 2015 版，新增双向通信基础框架，为 V2G 做准备 |
| **GB/T 33994-2017** | 电动汽车充换电设施电能质量评估技术规范 | 并网质量 | 2017 | 谐波、电压偏差、三相不平衡评估 |
| **GB/T 33589-2017** | 电动汽车车网协调通用技术要求 | V2G通用 | 2017 | 最早针对 V2G 的国标，框架性文件 |
| **GB/T 34855-2017** | 电动汽车充放电系统技术标准体系 | V2G标准体系 | 2017 | 定义 V2G 标准框架 |
| **GB/T 34132-2017** | 电动汽车充放电术语 | 术语 | 2017 | 统一 V2G 相关术语 |
| **GB/T 34657.1-2017** | 传导充电系统 第1部分：通用要求 | 电气 | 2017 | 直流/交流充电接口标准 |
| **GB/T 40429-2021** | **能源互联网与电动汽车互动规范** | 系统级 | 2021 | **V2G 核心国标**：系统架构、互操作、安全要求 |
| **GB/T 28142-2023** | 电动汽车非车载传导式充电设备技术条件 | 设备 | 2023 | 充电机/桩技术规范 |
| **GB/T 29785-2013** | 电动汽车 非车载传导式充电机与电池之间的通信协议 | 车-桩 | 2013 | 早期充电通信协议（已基本被 GB/T 27930 取代） |

### 3.2 重点解读：GB/T 40429-2021（能源互联网与电动汽车互动规范）

这是中国 **V2G 核心国标**，规定了：

- 系统架构：车辆 ↔ 充电桩 ↔ 运营平台 ↔ 电网
- 互操作要求：通信协议、数据结构、接口规范
- 能量双向流动：充电/放电模式切换
- 安全要求：电气隔离、漏电保护、紧急停机
- 计量要求：双向电表、电量清分

**适用场景**：国网/南网 V2G 项目验收必对标此标准。

### 3.3 GB/T 27930-2023（2023新版）

- 替代了 2015 版
- 新增了数字通信框架，支持未来 V2G 的双向信息交互
- 但 **尚未完全实现功率反向控制**，V2G 功率调度仍需上层 EMS/平台补充

## 八、关键建议

1. **如果要做国网/南网 V2G 项目**：首要对标 GB/T 40429-2021 + Q/GDW 11778-2017（国网）/南网等效标准
2. **充电桩出口欧美**：ISO 15118-20 + OCPP 2.0.1 + IEEE 2030.5 组合
3. **车端研发**：ISO 15118-20 + GB/T 27930-2023 双栈（中国市场）
4. **并网安全**：无论哪个市场，IEEE 1547-2018 或国标 GB/T 33593 都是电力公司的硬性要求
