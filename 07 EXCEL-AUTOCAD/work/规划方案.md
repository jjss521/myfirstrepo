# 配电柜系统图自动生成程序 — 规划方案

## 1. 目标与范围

将一份描述配电柜各回路参数的 Excel 表格导入程序，程序读取回路数据后，在 AutoCAD 中用预先建好的图块（Block）按电气设计规范自动拼接，生成完整的配电系统图（单线图/系统图）。

**输入**：配电柜回路 Excel 表格（包含断路器、电缆、负荷值、电流、变频器等字段）。
**输出**：AutoCAD DWG 文件，含完整配电系统图。
**用户**：电气设计工程师。

## 2. 总体架构

```mermaid
flowchart LR
    A["Excel 回路表"] -->|读取解析| B["数据层"]
    B -->|校验/映射| C["业务逻辑层"]
    C -->|调用 AutoCAD API| D["AutoCAD 绘图引擎"]
    D -->|输出| E["配电系统图 DWG"]

    F["图块库 DWG"] -->|加载 Block| D
    G["配置规则"] -->|驱动| C
```

分层说明：
- **数据层** — Excel 解析、字段映射、回路数据标准化
- **业务逻辑层** — 回路排序、图块选择、布局计算、连接线生成、标注
- **绘图引擎层** — AutoCAD COM/.NET 互操作，图块引用插入与属性编辑

## 3. 技术选型

| 层次 | 方案 | 理由 |
|---|---|---|
| 语言 | Python 3.10+ | 生态丰富，Excel 处理成熟，与 AutoCAD COM 互操作方便 |
| Excel 读取 | `openpyxl` + `pandas` | `openpyxl` 可读 .xlsx 且能处理合并单元格、样式；`pandas` 用于数据清洗和批量映射 |
| AutoCAD 互操作 | `pyautocad`（封装 `comtypes`） | 轻量，直接调用 AutoCAD ActiveX/COM，适合图形插入、属性编辑 |
| 图块管理 | 外部 DWG 图块库 | 断路器、电缆、变频器等预制成带属性的 DWG Block |
| 配置/规则 | YAML 或 JSON | 回路类型→图块名映射、排序规则、间距参数 |
| GUI/CLI | 命令行 + 可选简易 Tkinter/PySide GUI | 先 CLI 完成核心，再封装操作界面 |
| 测试 | `pytest` + mock AutoCAD 对象 | 数据层和逻辑层可脱离 AutoCAD 单独测试 |

### 3.1 方案对比：pyautocad vs .NET SDK vs AutoLISP

| 维度 | pyautocad (COM) | .NET SDK (C#) | AutoLISP |
|---|---|---|---|
| 上手难度 | 低 | 中高 | 中 |
| 调试便利性 | 高（Python REPL） | 中（需 VS） | 低 |
| Excel 处理能力 | 极强 | 需额外库 | 弱 |
| 性能 | 中等，适合百级回路 | 高 | 中等 |
| 部署 | 需安装 pyautocad/comtypes | 需编译 DLL，CAD 加载 | 脚本加载 |
| **推荐** | ✅ 主方案 | 备选（性能瓶颈时迁移） | 不选 |

## 4. Excel 数据模板设计

### 4.1 表结构（建议）

Sheet 名称：`回路清单`

| 列 | 字段 | 类型 | 说明 |
|---|---|---|---|
| A | 回路编号 | str | 如 L1, L2, PE1 |
| B | 回路名称 | str | 用途描述 |
| C | 负荷类型 | str | 照明/动力/空调/变频 |
| D | 额定功率(kW) | float | |
| E | 额定电流(A) | float | |
| F | 断路器型号 | str | 如 NSX100N |
| G | 断路器极数 | int | 1P/2P/3P/4P |
| H | 脱扣器额定电流(A) | float | |
| I | 电缆型号 | str | 如 YJV-4×25+1×16 |
| J | 电缆截面积(mm²) | float | |
| K | 变频器型号 | str | 如有 |
| L | 变频器功率(kW) | float | 如有 |
| M | 备注 | str | |

### 4.2 容错设计

- 断路器/变频器列为可选空值，空值表示该回路无对应元件
- 自动检测列名（支持中英文别名），映射表可配置
- 无效/缺失必填字段的行跳过并记录错误日志

## 5. 图块库规范

### 5.1 图块清单

在图块库 DWG 中预定义以下 Block：

- `CB_XXX` — 断路器图块（按型号细分，如 CB_NSX100N、CB_NSX160N），带属性 `RATED_CURRENT`、`POLES`、`CIRCUIT_ID`
- `CABLE_XXX` — 电缆标注图块，带属性 `TYPE`、`SECTION`
- `VFD_XXX` — 变频器图块，带属性 `MODEL`、`POWER`
- `LOAD_LABEL` — 负荷标注块，带属性 `POWER`、`CURRENT`
- `BUS_CONNECT` — 母线连接点

### 5.2 图块属性规范

```
Block: CB_NSX100N
  Attributes:
    CIRCUIT_ID    (可见)
    RATED_CURRENT (可见)
    POLES         (可见)
```

### 5.3 图块插入点约定

所有图块插入点统一为图块左下角或顶部母线连接点，保证布局一致性。

## 6. 布局算法

### 6.1 整体结构

```
母线 BUS
│
├── 回路1 ──┬── CB 断路器 ──┬── CABLE 电缆标注
│           │               │
│           │               └── LOAD 负荷标注
│           │
│           └── VFD 变频器（若有）
│
├── 回路2 ── ...（同上）
│
├── ...
│
└── 回路N
```

### 6.2 布局参数

- 水平方向：左→右 母线 → 断路器 → 变频器（如有）→ 电缆 → 负荷
- 垂直间距：每回路 60-80 绘图单位
- 水平间距：每元件 30-50 绘图单位
- 回路排列：按回路编号或负荷类型分组从上到下排列
- 母线走向：左侧垂直母线，各回路水平引出

### 6.3 布局流程伪码

```
Data=[]
Sorted_Circuits = sort(Data, by=circuit_id)
Y = TOP_Y
for circuit in Sorted_Circuits:
    X = BUS_X + OFFSET
    insert_block(BUS_CONNECT, BUS_X, Y)
    X += STEP
    cb_block = select_cb_block(circuit.breaker_model)
    insert_block(cb_block, X, Y, attrs={...})
    X += STEP
    if circuit.vfd_model:
        insert_block(VFD_XXX, X, Y, attrs={...})
        X += STEP
    insert_block(CABLE_XXX, X, Y, attrs={...})
    X += STEP
    insert_block(LOAD_LABEL, X, Y, attrs={...})
    draw_line(BUS_X, Y, X)       # 母线到最后一个元件的连线
    Y -= VERTICAL_SPACING
```

## 8. 核心模块详设

### 8.1 `excel_reader.py`

- 用 `openpyxl` 打开工作簿，读取 `回路清单` Sheet
- 首行识别列名（含中英文别名映射表）
- `pandas.DataFrame` 做清洗：去空行、类型转换、缺省值填充
- 输出 `List[Circuit]`

### 8.2 `data_model.py`

```python
@dataclass
class Circuit:
    circuit_id: str
    name: str
    load_type: str
    rated_power_kw: float
    rated_current_a: float
    breaker_model: Optional[str]
    breaker_poles: Optional[int]
    breaker_trip_current_a: Optional[float]
    cable_type: Optional[str]
    cable_section_mm2: Optional[float]
    vfd_model: Optional[str]
    vfd_power_kw: Optional[float]
    remark: Optional[str]
```

### 8.3 `block_mapper.py`

- 维护 `config.yaml` 中的映射表：`{断路器型号: 图块名}`
- 支持通配符/前缀匹配
- 未匹配到的型号记录警告并使用默认图块

### 8.4 `layout_engine.py`

- 读入回路列表，按负荷类型分组、回路编号排序
- 计算每个图块的绝对坐标
- 输出 `List[Placement]`（图块名 + 坐标 + 属性字典）

### 8.5 `cad_drawer.py`

- 初始化 `pyautocad.Autocad` 连接
- `load_block_library(block_dwg_path)` — 将外部图块库 DWG 的 Block 定义导入当前文档
- `insert_block(placement)` — 插入图块引用并设置属性
- `draw_polyline(points)` — 画母线、连接线
- `add_text(position, content, height)` — 添加文字标注
- 异常安全：所有 AutoCAD 操作包裹 try/except，异常写入日志而不是崩溃

### 8.6 `config.yaml` 示例

```yaml
excel:
  sheet_name: "回路清单"
  column_aliases:
    回路编号: circuit_id
    断路器型号: breaker_model
    # ...

block_mapping:
  breaker:
    "NSX100N": "CB_NSX100N"
    "NSX160N": "CB_NSX160N"
    "DEFAULT": "CB_GENERIC"
  vfd:
    "ATV320": "VFD_ATV320"
    "DEFAULT": "VFD_GENERIC"

layout:
  start_x: 0
  start_y: 297          # A3 高度
  bus_x: 50
  vertical_spacing: 70
  horizontal_step: 40
  drawing_size: "A3"
```

## 10. 关键风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|---|---|---|---|
| AutoCAD 版本 COM 接口不兼容 | 中 | 高 | 支持多版本 API 适配层，运行时检测版本 |
| 图块库 DWG 缺失或不规范 | 高 | 中 | 程序启动时校验图块库完整性，缺失给出明确提示 |
| Excel 格式不统一 | 中 | 中 | 提供标准模板，列名模糊匹配，严格校验输出错误清单 |
| pyautocad 性能不足（大量回路） | 低 | 低 | 监控插入耗时，必要时迁移到 .NET SDK 方案 |
| AutoCAD 未安装或未运行 | 中 | 中 | 启动时检测 AutoCAD 进程，无连接则提示并退出 |

## 11. 关键决策点（待确认）

1. **AutoCAD 版本**：目标版本是哪个（AutoCAD 2018/2020/2024 等）？这直接影响 COM ProgID。
2. **图块库来源**：是否已有现成的图块 DWG？还是需要从零创建？
3. **图纸幅面**：默认 A3 还是有多种幅面需求？
4. **回路数量级**：典型回路数是几十、几百还是上千？决定性能策略。
5. **运行方式**：单文件处理还是批量处理多个 Excel？
6. **配电系统类型**：低压配电柜（400V）还是中压？不同电压等级图块和符号体系不同。
